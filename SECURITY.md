# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 3.1.x   | Yes       |
| 3.0.x   | Yes (security fixes only) |
| < 3.0   | No        |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report vulnerabilities privately using one of the following methods:

- **GitHub Security Advisories** (preferred): Navigate to the repository's
  [Security tab](https://github.com/MRJR0101/Code-Normalizer-Pro/security/advisories/new)
  and open a private advisory.
- **Email**: Contact the maintainer directly at rawlsjrm@gmail.com with the
  subject line `[SECURITY] code-normalizer-pro`.

### What to include

- Description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected version(s)
- Any suggested fix, if you have one

### Response timeline

- **Acknowledgement**: within 48 hours of receipt
- **Initial assessment**: within 5 business days
- **Fix and release**: depending on severity, within 14–30 days

You will be credited in the release notes unless you prefer to remain anonymous.
